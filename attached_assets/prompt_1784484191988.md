Anda adalah Senior Laravel Developer, Backend Engineer, dan UI/UX Developer.

Project ini adalah **Server Pusat Monitoring & Update** untuk aplikasi **Absensi QR Sekolah** yang sudah berjalan di banyak VPS sekolah.

---

## Konteks Sistem

Setiap sekolah menjalankan aplikasi **Absensi QR Sekolah** (Laravel 13, PHP 8.4) di VPS mereka sendiri.

Aplikasi sekolah sudah diprogram untuk:

1. Mengirim **heartbeat** setiap jam ke `POST /api/heartbeat`
2. Mengecek **versi terbaru** sekali sehari ke `GET /api/releases/latest`

Server pusat ini adalah endpoint yang menerima request tersebut, menyimpannya ke database, dan menyediakan dashboard untuk developer (Anda sendiri) memantau seluruh instalasi sekolah.

## Kontrak API yang WAJIB Dipatuhi

> ⚠️ Format ini sudah hardcoded di aplikasi sekolah. Jangan ubah struktur request/response.

### 1. `POST /api/heartbeat`

**Headers dari aplikasi sekolah:**
```
X-Installation-ID: {installation_id}     ← string 32 hex chars
X-App-Version: {version}                 ← contoh: "1.0.0"
Content-Type: application/json
Accept: application/json
```

**Body JSON dari aplikasi sekolah:**
```json
{
  "installation_id": "7c2cf4c7172b189e7b7f76db57361656",
  "app_version":     "1.0.0",
  "app_name":        "Absensi QR Sekolah",
  "app_url":         "https://absensi.sman1contoh.sch.id",
  "php_version":     "8.4.2",
  "db_driver":       "pgsql",
  "wa_online":       true,
  "update_channel":  "stable",
  "timestamp":       "2026-07-19T16:55:00+00:00"
}
```

**Response yang diharapkan (200 OK):**
```json
{
  "status": "ok",
  "received_at": "2026-07-19T16:55:01Z"
}
```

**Perilaku server:**
- Jika `installation_id` belum dikenal → simpan sebagai instalasi baru (auto-register).
- Jika sudah dikenal → update data yang ada.
- Simpan setiap heartbeat ke tabel log (riwayat) — bukan hanya update last-seen.
- Rate limit: maks **20 request per jam per `installation_id`**.
- Response harus **< 300ms** — tidak ada operasi berat di sini.

---

### 2. `GET /api/releases/latest`

**Headers dari aplikasi sekolah:**
```
X-Installation-ID: {installation_id}
X-App-Version: {version}
Accept: application/json
```

**Query params:**
```
?version=1.0.0&channel=stable
```

**Response yang diharapkan (200 OK):**
```json
{
  "latest_version": "1.2.0",
  "released_at":    "2026-07-15",
  "channel":        "stable",
  "category":       "feature",
  "title":          "Pembaruan Juli 2026",
  "notes":          "## Fitur Baru\n- Tambah ekspor PDF\n- Fix live monitor mobile",
  "mandatory":      false,
  "announcement": {
    "title":    "Maintenance Server",
    "message":  "Server pusat maintenance 20 Juli 02.00–04.00 WIB",
    "priority": "info",
    "until":    "2026-07-20T04:00:00Z"
  }
}
```

> Jika tidak ada update (versi sama dengan yang dikirim), kembalikan `latest_version` sama dengan `version` query param dan `mandatory: false`.
> Jika tidak ada pengumuman aktif, kembalikan `"announcement": null`.
> Hanya kembalikan rilis dengan `is_active = true` dan `channel` yang sesuai.

## Keamanan API

Karena tidak ada sistem lisensi, gunakan pendekatan berikut:

### Rate Limiting

Di `RouteServiceProvider` atau `bootstrap/app.php`:
```php
RateLimiter::for('heartbeat', function (Request $request) {
    $id = $request->input('installation_id') ?: $request->header('X-Installation-ID');
    return Limit::perHour(20)->by($id ?: $request->ip())
        ->response(fn() => response()->json(['error' => 'Too many requests'], 429));
});
```

### Validasi Request

Untuk `POST /api/heartbeat`:
- `installation_id` wajib, string, maks 64 karakter, regex `[a-f0-9]`
- `app_version` wajib, string, maks 20 karakter
- `timestamp` opsional — jika ada, tolak jika lebih dari **10 menit** di masa lalu atau masa depan (anti-replay sederhana)
- Field lain opsional — jangan strict, sekolah mungkin kirim field ekstra

Untuk `GET /api/releases/latest`:
- Tidak ada validasi ketat — endpoint ini bisa diakses publik
- Log `X-Installation-ID` jika ada untuk analitik

### HTTPS

- Pastikan aplikasi ini di-deploy dengan HTTPS (Nginx + Let's Encrypt)
- Di production, set `FORCE_HTTPS=true` dan redirect HTTP ke HTTPS di Nginx
